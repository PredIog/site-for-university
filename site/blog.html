Обсуждение идей проекта

На первой встрече команды мы собрались, чтобы обсудить основную концепцию проекта. Было рассмотрено несколько вариантов решения проблемы переноса медиатек между стриминговыми сервисами.

Основные вопросы, которые мы обсудили:

  • Какие стриминговые платформы поддерживать в первую очередь (Spotify, Яндекс Музыка, VK Музыка)
  • Какой формат переноса данных будет наиболее удобным для пользователя
  • Возможность объединения коллекций с нескольких платформ в одну

В итоге мы утвердили основные требования к функционалу и распределили зоны ответственности между участниками.

9 февраля 2026

Обсуждение архитектуры и технических решений

На этой неделе команда сосредоточилась на проектировании архитектуры будущего инструмента. Мы определили стек технологий и структуру взаимодействия между компонентами.

Ключевые решения:

  • Использование REST API для взаимодействия со стриминговыми сервисами
  • Формат хранения данных — JSON для сохранения структуры метаданных
  • Клиент-серверная архитектура с локальным хранением кэша

Также мы проанализировали API выбранных платформ и оценили сложность интеграции с каждой из них.

16 февраля 2026

Старт разработки

Мы начали кодить! Первая версия инструмента постепенно обретает форму. На данном этапе реализована базовая авторизация через API и получение списка плейлистов пользователя.

Что уже сделано:

  • Подключение к API Яндекс Музыки
  • Получение структуры плейлистов (названия треков, исполнители, альбомы)
  • Сохранение полученных данных в локальный кэш

В ближайших планах — реализация экспорта плейлистов из Spotify и импорта в целевой сервис.

2 марта 2026

Продолжение разработки

Разработка идёт полным ходом! Мы успешно реализовали функционал переноса плейлистов между сервисами и приступили к тестированию.

Текущие достижения:

  • Готов модуль переноса данных из Spotify в Яндекс Музыку
  • Реализована функция объединения нескольких плейлистов в один
  • Написаны юнит-тесты для проверки целостности метаданных
  • Проведено первое интеграционное тестирование

Обнаруженные баги уже исправлены. На следующей неделе планируем добавить поддержку VK Музыки и начать подготовку к бета-тестированию.
